什麼是資料庫系統 ?

資料庫系統可以說是，資料庫 Database存放資料的地方，和 資料庫系統 Database Management System 控制資料存取和權限組成。

資料庫 Database

資料存在你我的生活之中，只是把資訊給表格化像 excel 的表格，但根據資料庫的設計，邏輯會更加複雜

資料庫系統

管理資料庫的應用程式就叫資料庫系統

其中又分成關聯式資料庫 Relational Database 又稱 RDBMS 像是MySQL、Miceosoft SQL Sever、和非關聯式資料庫 Non-SQL 像是 mongodb ，有這兩種資料庫系統。

非關聯式資料庫 NoSQL

跟RDBMS相比，彈性高、速度快，資料通常儲存在 JSON 或類似的資料結構裡，最大特色是 :
不要求資料同步，只要求最後結果一致。
不需要定義 schema，資料之間沒有關聯關係，有名的NoSQL有MongoDB、Redis等，且NoSQL為水平擴展(增加另一台雲端設備)，擴展成本較低。

適用情況 : NoSQL大多為開源軟體，所以適用沒什麼經費的小型專案，及開發初期需要頻繁變動資料庫時、或儲存大量資料時。

關聯式資料庫 RDBMS

特色如其名，資料與資料之間可以用 FK 做一對一 ( 1 :1 ) 、一對多 ( 1 : N )、多對多( M : N )的關聯，且好的資料庫會有 ACID 來確保完整性。

  • Atomicity 原子性 : 一次的交易 transaction 只能有兩種結果，全部完成 commit、全部不完成 rollback，不會卡一半。
  • Consistency 一致性 :transaction 中會驗證是否符合 schema 規範，不會讓不一樣的格式進入資料庫中。
  • Isolation 隔離性 : 多筆交易 transaction 可以同時進行，但不會在交易完成前互相干擾。
  • Durability 耐久性 : transaction 完成後對資料的修改，不會因為重新啟動或錯誤而改變。

適用情況 : 在系統資料龐大、商業邏輯複雜、存取嚴謹度要求高的需求下，使用關聯式資料庫應該是比較好的選擇。

關聯式資料庫也有以下幾個缺點 :

  • 費用較高
  • 邏輯複雜、維護成本比較高、弄不好會有死鎖 Deadlock 的問題。
  • 資料庫通常是產品運作的命脈，所以通常都會選效能最強的設備來當資料庫的伺服器，但後續擴展困難，通常式垂直擴展 (電腦硬體設備)

資料庫管理模組

一個關聯式資料庫通常包括以下四個模組

  • 儲存管理 Storage Manager
  • 查詢處理模組 Query Processor
  • 交易管理 Transaction Manger
  • 回復管理 Recovery Manger

Keys 關係鍵

  • 關聯式資料庫重要的組成部分，在 DBMS 中，主要的 Keys 有 :
  1. 主鍵 primaty key ( 簡稱PK )
    唯一的表示每一筆資料，一筆資料只能有一個主鍵，而且不能是空 Null 的，它也可以是連接到其他表格的PK。
  2. 外鍵 Foreign Key ( 簡稱FK )
    指向其他表格主鍵的欄位，確保兩張表格的關聯性，只能對應到 PK ( 避免有其他數值導致錯誤 )
  3. 自然鍵 Natural key
    真實生活中唯一確定一個事物的標誌，來當作主鍵 primary key 可被稱為 natural key ， ex : 身分證字號
  4. 代理鍵 Sirrogate key
    資料表格中，所有欄位都不適合當主鍵時，就會製作一個無意義的欄位來當作主鍵
  5. 複合主鍵 comopsite key
    一筆資料只能有一個主鍵，( 但可以由兩個以上的行組成PK )
